## 3.0方案

1. 按指标分类设计数据表（血压、血糖、血脂、血常规、肝功能、肾功能、尿常规、体重/BMI、睡眠、其他共约 10 类）。
2. 使用 LLM 将用户每天上传的文件统一解析为标准 JSON 指标结构。
3. 将 JSON 中各类指标按分类写入对应指标表，形成可查询的时间序列数据。
4. 对单一指标做时间序列分析（如血压、血糖、血脂等的趋势与波动评估）。
5. 计算综合风险指标（如 TG/HDL、LDL/HDL、TC/HDL、肝硬化与肾功能评分模型）。
6. 进行关联分析（如血脂与血糖、睡眠质量与血压/血糖等的相关性评估）。
7. 基于时间序列与综合指标生成多种图表（折线图、散点图、风险趋势图等）。
8. 由分析结果与图表自动生成面向患者和医生的双版本健康管理报告。

## 3.0问题

### 一、结构与数据层问题

1. **LLM 生成 JSON 是否足够稳定、可重复**
2. **指标名称、单位、参考区间是否会不一致**
3. **时间字段是否可靠（来源混杂、补标、缺失）**
4. **异常值 / 错误值是否会直接进入时间序列**
5. **md → JSON 的二次解释是否会放大错误**

---

### 二、模型与分析层问题

6. **LLM 幻觉一旦进入表结构，是否可被发现**
7. **不同模型版本生成的数据是否可共存**
8. **历史数据是否允许被“重新解释”**
9. **复合指标和风险模型是否等于冻结医学假设**
10. **关联分析是否会产生伪相关**

---

### 三、系统演进问题

11. **指标分类（10 类）未来变化如何兼容**
12. **schema 变更是否会破坏历史数据**
13. **是否存在无法回滚的数据污染路径**

---

### 四、责任与语义问题

14. **患者版 / 医生版报告是否隐含责任切换**
15. **系统结论是否会被误认为医学诊断**
16. **统计结论与医学结论的边界是否模糊**

---

### 五、工程现实问题

17. **3.0 是否反向要求你重做 2.0**
18. **调试与验证 LLM 输出的成本是否可控**
19. **错误一旦进入时间序列，是否可修复**
20. **你是否能解释“某个结论是怎么来的”**
